Baseball score processing

Background

A game of baseball or softball is composed of innings. Each inning has two halves. In the first half of an inning, the visiting team gets a chance to score runs; in the second half, the home team takes its turn to try to score. The visiting team's total score is the sum of the number of runs in the first half of each inning; similarly, the home team's total score is the sum of the number of runs in the second half of each inning.

A baseball game normally consists of nine innings. There are two exceptions.

  • When the home team's score exceeds the visiting team's score after completing the first half of the ninth inning, the game ends immediately with 8½ innings;
  • When at least nine innings have been played and the score is tied, the game goes into extra innings. Another inning is played; if the score is still tied, another inning is played, and so on.

Problem

Scorer.java, already in your imported project, is a framework for a program to read half-inning scores from a baseball game and print the total number of runs scored by each team as well as information about which team won.

This code uses Java methods to perform different sub-problems. The comments for each method are intended to help you understand what each sub-problem is and how the method is supposed to solve it. Included in the comments are descriptions of your tasks in this exercise.

You are to complete the framework and make the code work. You'll change each of the methods in Scorer.java.
