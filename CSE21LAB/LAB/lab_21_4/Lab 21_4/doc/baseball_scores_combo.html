Baseball score processing, part 3 (Testing & Overloading)

Problem

We have two different ways to summarize the outcome of a baseball game. First is just a statement of who won and in how many innings. Second way is more detailed with the scores for each half innings. We will now add the capability for the users to choose which type of output they want to see. Both of these behavior belongs to result so how can we make multiple version of it? Overloading is an easy to enable putting different behaviors inside the same method name but with different signatures. We will tweak our method design so we are no longer reliant on class variables to hold all the information. We will move the declaration inside each of the methods and pass the information around instead. This makes the solution a little more complicated but it allows for testing each individual methods. Unit tests for this lab are made for you already in ScorerComboTest.java.
